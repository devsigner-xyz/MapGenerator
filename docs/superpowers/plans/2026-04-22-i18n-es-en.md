# Internationalization ES/EN Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add English and Spanish internationalization to the overlay and landing, persist the selected language in UI settings, and expose a language selector in `Settings > UI`.

**Architecture:** Introduce a lightweight typed i18n layer with `es` and `en` message catalogs, a shared `I18nProvider` for the two React roots, and a reusable `translate` helper for future non-React surfaces. Keep the current route slugs unchanged and leave legacy `dat.gui` controls out of scope for this first iteration.

**Tech Stack:** React 19, React Router 7, TypeScript, Vitest, Playwright, shadcn/ui.
